# v0.2.x-summary.md

## v0.2.0 (2026-05-15)

Seven milestones shipped in one release series off `release/v0.2.0`.

**M1 — OR operator in query language**
Extended the query grammar from single-level AND-chains to full AND/OR with
correct precedence: AND binds tighter than OR, each AND-group is parenthesized
in the emitted SQL (`WHERE (a=? AND b=?) OR (c=?)`). The AST was reshaped from
`QueryNode::And(Vec<Clause>)` to `QueryNode::Or(Vec<AndGroup>)` — a breaking
public API change for `logdive-core` library consumers.

**M2 — logfmt and plain-text ingestion**
Added `--format json|logfmt|plain` to `logdive ingest`. Introduced the
`LogFormat` enum, the `parse_line(format, line)` dispatcher, and two new parser
submodules (`parsers::logfmt`, `parsers::plain`). Also added `--timestamp-now`
as the user-controlled escape hatch for timestamp-less formats. Another breaking
change: `parse_line` signature changed from `(line: &str)` to
`(format: LogFormat, line: &str)`.

**M3 — Follow mode**
Added `logdive ingest --follow` (Unix only). Uses `notify` 6.1 for filesystem
events and `ctrlc` 3.5 for clean Ctrl-C shutdown. `FileTailer` in
`crates/core/src/follow.rs` handles rotation (inode/device change detection via
`MetadataExt`) and truncation. The CLI watch loop is fully synchronous — no
tokio dependency was added to the CLI crate.

**M4 — prune subcommand + LOGDIVE_DB env var**
Added `logdive prune --older-than <duration>` and `--before <datetime>`, with
interactive confirmation and `--yes` bypass. Added `LOGDIVE_DB` environment
variable fallback on the global `--db` flag in both CLI and API binaries.

**M5 — API capability endpoints + CORS**
Added `GET /version` returning `{version, formats, capabilities}` as a
compile-time-constant endpoint (never touches DB). Added `--cors-origins` to
`logdive-api` with wildcard/specific-origin validation. Wired `tower-http`
`CorsLayer` into the router. The `build_router` function gained a
`cors_origins: Vec<HeaderValue>` parameter — breaking for integration test
consumers that constructed routers directly.

**M6 — Docker image + multi-arch**
Multi-stage `Dockerfile` using cargo-chef layer caching. Both binaries in one
image; `logdive-api` is the default entrypoint, `logdive` accessible via
`--entrypoint`. GHA workflow for `linux/amd64` + `linux/arm64` via QEMU.
`HEALTHCHECK` on `GET /version`. Non-root `logdive` user (UID/GID 1000).
`ENV LOGDIVE_DB=/data/index.db` and `ENV LOGDIVE_API_HOST=0.0.0.0` set
container-native defaults. `logdive-api` auto-creates empty index on first run.

**M7 — Release polish**
CHANGELOG, version bump, README updates, `scripts/prerelease-check.sh`,
`scripts/check-binary-size.sh`, GitHub issue templates, FUNDING.yml, OSS
metadata (description, keywords, categories in Cargo.toml), dual MIT/Apache-2.0
license headers.

**Performance baseline (from README, measured on Acer Nitro 5, Linux):**

| Operation | Throughput / Latency |
|---|---|
| Ingestion, batched insert (10k rows) | ~210k lines/sec |
| Ingestion, parse + insert end-to-end (10k rows) | ~166k lines/sec |
| Query on known field, empty result (100k rows) | ~17 μs |
| Query on known field, 25% match (100k rows, LIMIT 1000) | ~39 ms |
| Query on JSON field, 25% match (100k rows, LIMIT 1000) | ~3.6 ms |
| Query on JSON field, 0% match — full scan (100k rows) | ~68 ms |
| CONTAINS full-table scan (100k rows) | ~36–40 ms |
| 3-clause AND chain (100k rows) | ~22 ms |

## v0.2.1 (2026-06-01)

Five hardening milestones (H1–H5) shipped together in one squash-merged PR
from `chore/v0.2.1/hardening` + `chore/v0.2.1/docs`.

**H1 — Security tests** (10 new tests in `crates/core/tests/security.rs`)
SQL injection via field name: tokenizer rejects `'`, `;`, Unicode lookalikes
(U+2019). SQL injection via value: bound parameters prevent DROP TABLE and
tautology payloads. LIKE wildcard escaping: `_` and `\` in `CONTAINS` queries
match literally. Resource exhaustion: 1,000-disjunct OR query parses without
stack overflow (parser is iterative, not recursive); 10 MB raw line ingests
without panic or OOM.

**H2 — Functional tests** (28 new tests across 7 suites)
Property-based tests with `proptest`: arbitrary input never panics; valid
equality queries produce single-group ASTs; OR disjunct count matches input;
printable ASCII in quoted values never panics. Cross-format dedup test suite.
Concurrent CLI ingest test (two processes, same DB, no corruption). Parser edge
cases: UTF-8 BOM rejection, deeply nested objects, whitespace preservation.
Time-range boundary tests. Follow-mode: file-deleted-after-open returns Ok([]).
API integration: limit larger than result set, contains operator, since
time-range, CORS preflight, raw field present. Prune boundary precision (1
second), idempotency.

**H3 — Supply-chain hardening**
`Cargo.lock` tracked for reproducible builds. `deny.toml` with license allowlist
(MIT/Apache-2.0/BSD/ISC/CC0-1.0/Zlib/BSL-1.0 and variants), RUSTSEC advisory
denial, crates.io-only source policy. `scripts/audit.sh` for `cargo-audit`.
`scripts/sbom.sh` for CycloneDX SBOM via `cargo-cyclonedx`. Daily
`.github/workflows/audit.yml`. CI `lint` job now runs `cargo deny check`
(merge-blocking); CI `permissions: contents: read` tightened.

**H4 / H5 — Performance + allocation improvements** (listed in CHANGELOG as "Changed")
`LogEntry::with_tag` signature changed from `Option<String>` to `Option<&str>`,
eliminating one `String` clone per ingested entry. `entry_to_json_string` in the
API now uses `serde_json::to_string(&entry)` directly instead of constructing a
`json!` macro value with `entry.fields.clone()`, eliminating an O(fields) heap
allocation per HTTP response row.

**Test count:** 330 (v0.2.0) → 386 (v0.2.1), net +56.

## v0.3.0 (2026-06-05)

Five milestones (M1–M5) shipped in one release series off `release/v0.3.0`.

**M1 — Parenthesised query groups**
Added `Clause::Group(Box<QueryNode>)` to the query AST. Parser extended with
`parse_primary()` that recognises `(` and recurses into `parse_or_expr()`,
closing on `)`. Executor wraps group clauses in SQL `(sub_expression)`. Grammar
is now: `clause := field OP value | field CONTAINS string | "(" or_expr ")"`.

**M2 — CLI pagination and output flag rename**
`--format` on `logdive query` renamed to `--output` (`pretty|json`) to avoid
confusion with `ingest --format` (input format). **Breaking CLI change.**
`QueryOptions { limit: Option<usize>, offset: Option<usize> }` struct introduced
in `logdive-core`; replaces bare `limit: Option<usize>` on `execute()` /
`execute_at()`. **Breaking library change.**
`--offset N` added to `logdive query` for page navigation (default 0).

**M3 — HTTP pagination**
`GET /query?offset=<n>` parameter added to `logdive-api`. Absent or `0` starts
from the first result. `QueryParams` struct gains `offset: Option<usize>`. Three
new integration tests added.

**M4 — Case-insensitive level queries**
Expression index `CREATE INDEX IF NOT EXISTS idx_level_norm ON log_entries(lower(level))`
added to schema — idempotent, existing databases pick it up on next `Indexer::open()`.
`translate_clause()` in `executor.rs` now routes `level` field comparisons and
`CONTAINS` through `lower(level) = ?` with Rust-lowercased bind values.
`level=ERROR`, `level=Error`, `level=error` all hit the same rows.

**M5 — Distroless Docker runtime**
Runtime stage changed from `debian:bookworm-slim` to
`gcr.io/distroless/cc-debian12:nonroot` (uid 65532, no shell, no curl, no root).
`/data` directory created in the builder stage and `COPY`-ed to runtime since
distroless has no shell for `RUN mkdir`.
`logdive-api --health-check` flag added: TCP-connects to own port via stdlib
`TcpStream`; exits 0/1. `HEALTHCHECK CMD` updated to use this flag.

**Test count:** 386 (v0.2.1) → 417 (v0.3.0), net +31.

## Breaking changes log

Cumulative list of public API breaks in `logdive-core` since v0.1.0:

| Version | What broke | Migration |
|---|---|---|
| v0.2.0 | `QueryNode::And(Vec<Clause>)` removed; replaced by `QueryNode::Or(Vec<AndGroup>)` | Match on `QueryNode::Or(groups)` and iterate groups |
| v0.2.0 | `parse_line(line: &str)` → `parse_line(format: LogFormat, line: &str)` | Pass `LogFormat::Json` as first arg for previous behavior |
| v0.2.0 | `build_router(state)` → `build_router(state, cors_origins: Vec<HeaderValue>)` | Pass `vec![]` for CORS-disabled behavior |
| v0.2.1 | `LogEntry::with_tag(Option<String>)` → `with_tag(Option<&str>)` | Pass `.as_deref()` if you had an `Option<String>` |
| v0.3.0 | `execute(query, conn, Option<usize>)` → `execute(query, conn, QueryOptions)` | Wrap limit in `QueryOptions { limit, offset: None }` |
| v0.3.0 | `execute_at(query, conn, Option<usize>, now)` → `execute_at(query, conn, QueryOptions, now)` | Same as above |
| v0.3.0 | CLI `logdive query --format` renamed to `--output` | Update any scripts using `--format pretty` or `--format json` |
